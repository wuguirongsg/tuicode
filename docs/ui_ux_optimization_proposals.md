# TuiCode UI/UX 优化建议文档 v1.0

本文档基于对 TuiCode v0.2 版本的 UI 架构、交互实现及视觉风格的深度评估，提出了针对性的优化方案，旨在进一步提升 AI 协同编程的效率与体验。

---

## 1. 布局灵活性优化 (Layout Flexibility)

### 1.1 右侧工具栏动态调宽
*   **现状**：`RightPanel` 宽度固定为 28 列，导致长文件名被截断，Git 状态显示拥挤。
*   **优化建议**：
    *   在 `RightPanel` 左侧引入 `ResizeHandle`（类似于 `TerminalStrip` 的顶部手柄）。
    *   允许用户左右拖拽以动态调整文件树和 Git 视图的显示区域。
*   **价值**：适配不同分辨率的终端环境，提升复杂目录结构的浏览体验。

### 1.2 浮窗拼贴模式 (Tiling Mode)
*   **现状**：目前依赖手动排列或 3 种固定预设，窗口堆叠时难以快速定位。
*   **优化建议**：
    *   实现 `Ctrl+\` 快捷键，触发 `tiling` 逻辑：自动计算当前所有可见浮窗的最优网格排列。
    *   移除拼贴模式下的窗口边框装饰，以最大化代码显示面积。
*   **价值**：提供类似于 `tmux` 或 `i3wm` 的高效率平铺体验，同时保持浮窗的灵活性。

## 2. 交互效率增强 (Interaction Efficiency)

### 2.1 Diff 预览的自适应重绘
*   **现状**：`DiffPreviewWindow` 在窗口缩放时不会重新计算 `split` 布局，导致内容错位。
*   **优化建议**：
    *   监听 `Resize` 事件，根据 `self.size.width` 重新调用 `_render_split`。
    *   支持 `Ctrl+U / Ctrl+D` 快速翻页，增强长 Diff 的审查速度。
*   **价值**：确保用户在不同大小的浮窗中都能准确审查 Agent 的代码变更。

### 2.2 任务栏状态同步
*   **现状**：`WindowTaskBar` 仅显示标题和最小化状态。
*   **优化建议**：
    *   **脏状态标记**：若编辑器有未保存修改，在任务栏标题旁显示 `*`。
    *   **Agent 活动指示**：若 Agent 正在 PTY 中持续输出，任务栏图标颜色产生呼吸律动。
*   **价值**：减少用户频繁切窗确认状态的操作，实现真正的“全景感知”。

---

## 3. 信息架构与感知度 (Information Density)

### 3.1 全局状态栏 (StatusBar) 增强
*   **现状**：目前仅显示版本号和 Agent 计数。
*   **优化建议**：
    *   **Git 分支显示**：在左侧增加当前仓库分支名（如 `main`）。
    *   **活动路径**：显示当前聚焦文件的相对路径。
    *   **快捷键动态提示**：根据当前聚焦组件（如编辑器 vs 终端）动态切换底栏的快捷键列表。
*   **价值**：将核心开发上下文提升至全局可见层级，减少认知负荷。

### 3.2 吉祥物 (Mascot) 深度集成
*   **现状**：吉祥物（Braille LED）主要是视觉点缀。
*   **优化建议**：
    *   **点击交互**：点击吉祥物弹出“快捷指令”菜单或“新手引导”。
    *   **多维度反馈**：根据 Agent 的响应速度或命令执行结果（成功/失败），吉祥物展示不同的微表情。
*   **价值**：增强产品的“生命感”和品牌记忆点，同时兼顾辅助交互。
